Docker CI/CD 需求分析

原来：

  • 手动构建 -> 编译、测试
  • 手动部署 -> 敲一串命令
  • 手动配置 -> 改写配置文件 ….

一个服务发布，就要走一边这个流程… 那么，10 边呢？ 一个月 10 边呢？ 一年呢？？？

很繁琐，严重影响了生产效率，而且很容易出错

研究 CI/CD 的期望：

  • 就是希望能做到一次提交(Master)，自动构建、部署一套服务
  • 构建流程可追踪、可查阅、可预览、可反馈
  • 能与 Bitbucket 集成
  • 整个流程最好都基于 Docker 实现(我要的这一套方案就是由一个个的 Docker 容器组成的)
  • 这个 CI/CD 平台最好有权限管理功能，因为你做好交付给别人使用时，不能保证他们不会乱点、乱按…
