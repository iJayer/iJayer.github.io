go build

go build 把源码包和相关的依赖包编译成一个可执行文件

1
2

usage: build [-o output] [-i] [build flags] [packages]
Run 'go help build' for details.

go build 的使用比较简洁，所有的参数都可以忽略，直到只有 go build, 这个时候意味着在当前目录下进行编译，下面几条命令是等价的：

1
2
3

go build 
go build .
go build main.go

以上三种写法，都是使用当前目录编译的意思。因此我们忽略了 packages, 所以自然就使用当前目录进行编译了。从这里我们可以推测出，go build 本质上是需要一个路径，让编译器可以找到哪些需要编译的 go 文件。 packages 其实是一个相对路劲，是相对于我们定义的 GOROOT 和 GOPATH 这两个环境变量的，所以有了 packages 这个参数后，go build 就可以知道那些需要编译的 go 文件了。

1

go build instance.golang.com/go-std-command/_go_build

这样的方式是指定包的方式，这样会明确我们要编译那个包。当然我们也可以用通配符

1

go build instance.golang.com/go-std-command/_go_build/...

... 三个点表示匹配所有字符串，这样 go build 就会编译 tools 目录下的所有包

知识点：Go 源码文件有三大类 - 命令源码文件：声明自己属于 main 代码包、包含无参数声明和结果声明的 main 函数 - 库源码文件：集中放置各种待被使用的程序实体（全局常量、全局变量、接口、结构体、函数等等） - 测试源码文件：xxx_test.go 用于对前两种源码文件中的程序实体的功能和性能进行测试

-o      指定输出的文件名, 可以带上路径: 例如，go build -o /d/setup/bin

-i      安装相应的包, 即 go build + go install 组合

-a      force rebuilding of packages that are already up-to-date.

-n      把需要执行的编译命令打印出来，但是不执行，这样就可以很容易的知道底层是如何运行的

-p n    指定可以并行可运行的编译数目，默认是CPU数目

-race   开启竞态条件的检测

-v      打印出来我们正在编译的包名

-work   打印出编译时生成的临时工作目录的路径，并在编译结束时保留它。在默认情况下，编译结束时会删除该目录。

-x      打印编译过程中执行的命令，其实就是和-n的结果类似，只是这个会执行

-asmflags
        arguments to pass on each `go tool asm` invocation.
        此标记可以后跟另外一些标记，如-D、-I、-S等。这些后跟的标记用于控制Go语言编译器编译汇编语言文件时的行为。

-buildmode
        此标记用于指定编译模式，使用方式如 -buildmode=default
（这等同于默认情况下的设置）。此标记支持的编译模式目前有8种。借此，我们可以控制编译器在编译完成后生成静态链接库（即.a文件，也就是我们之前说的归档文件）、动态链接库（即.so文件）或/和可执行文件（在Windows下是.exe文件
        
-compiler name 
        指定编译是使用的编译器，gccgo还是gc

-gccgoflags '[pattern=]arg list'
        传递参数给gccgo编译连接调用
        
-gcflags '[pattern=]arg list'
        传递参数给5g, 6g, 8g 调用

-installsuffix suffix
        a suffix to use in the name of the package installation directory,
        in order to keep output separate from default builds.
        If using the -race flag, the install suffix is automatically set to race
        or, if set explicitly(明确的), has _race appended to it. Likewise(同样) for the -msan
        flag. Using a -buildmode option that requires non-default compile flags
        has a similar effect.

-ldflags '[pattern=]arg list'
        rguments to pass on each `go tool link` invocation.

-linkshared
        link against shared libraries previously created with
        -buildmode=shared.

-pkgdir dir
        install and load all packages from dir instead of the usual locations.
        For example, when building with a non-standard configuration,
        use -pkgdir to keep generated packages in a separate location.

-tags 'tag list'
        a space-separated list of build tags to consider satisfied during the
        build. For more information about build tags, see the description of
        build constraints in the documentation for the go/build package.
        
        此标记用于指定在实际编译期间需要受理的编译标签（也可被称为编译约束）的列表。这些编译标签一般会作为源码文件开始处的注释的一部分，例如，在$GOROOT/src/os/file_posix.go开始处的注释为：
        
        // Copyright 2009 The Go Authors. All rights reserved.
        // Use of this source code is governed by a BSD-style
        // license that can be found in the LICENSE file.

        // +build darwin dragonfly freebsd linux nacl netbsd openbsd solaris windows

-toolexec 'cmd args'
        a program to use to invoke toolchain programs like vet and asm.
        For example, instead of running asm, the go command will run
        'cmd args /path/to/asm <arguments for asm>'.
